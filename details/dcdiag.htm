- Tool Overview

Category
Information Collection
Description
Analyzes and examines the status of the Domain Controller.
Example of Presumed Tool Use During an Attack
This tool is used to investigate problems with the Domain Controller and use the results to examine attack methods.

- Tool Operation Overview

Item Description
OS Windows Server
Belonging to Domain Not required
Rights Administrator
Service Active Directory Domain Services

- Information Acquired from Log

Standard Settings
  • Host
    • Execution history (Prefetch)
Additional Settings
  • Host
    • The fact that dcdiag.exe was executed and details of the executed test (Sysmon)

- Evidence That Can Be Confirmed When Execution is Successful

  • The successful execution of the tool cannot be determined from event logs, execution history, and so on. (If the extracted account information is saved, it can be considered that tool execution was successful).

- Remarks

  • This tool is used to examine the status of the Domain Controller. By entering correct authentication information at execution, it is possible to acquire information even from a client that does not participate in the domain.
  • In Event ID: 4689 (Process Termination), if dcdiag.exe was executed successfully, the return value will be 0 regardless of the success or failure of the examination. For this reason, it is not possible to confirm the success or failure from the log, except when the results were saved to a file with the "-f" option.
